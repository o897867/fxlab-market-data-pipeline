## 1. v2 总览与优先级

排序原则:**数据越现成、对用户越直接的越先做;数据越贵、不确定性越高的越后做。**

| 优先级 | 模块 | 核心价值 | 数据现成度 | 依赖 |
|---|---|---|---|---|
| 1 | 影响面板 | 回答"期权怎么影响正股" | 高(v1 管道已具备)| 无 |
| 2 | 时序特征 | IV Rank、墙的演变、财报 IV 塌缩 | 中(需攒快照)| **快照任务必须 v1 就启动** |
| 3 | 期限结构面板 | 市场觉得近期 vs 远期有事 | 高(扩展拉多个到期日)| 无 |
| 4 | Narrative 层 | 新闻情绪、电话会语气 | 低(数据贵)| 可复用 MacroPulse |
| 加分 | 预测准不准回看 | 把历史变成信任工具 | 中(需快照)| 同优先级 2 |
| 加分 | 多标的 / 自选股 | 从单票到 watchlist | 高 | 无 |

## 2. ⚠️ 关键依赖:快照任务必须在 v1 就启动

这是整个 v2 里**唯一一个「现在不做、以后补不回来」的东西**。

时序特征(优先级 2)和预测回看都依赖历史快照。InsightSentry 的历史链**不含已过期合约**,意味着你无法事后回填——只能从某一天起每天落一张当前快照,往后攒。

**行动**:v1 上线那天,就把下面两张 append-only 表的定时任务跑起来,**哪怕 v1 界面上完全不显示它们**。否则等你想做 IV Rank 时才从零攒,又要白等几个月。

```
fct_iv_snapshot   (append-only)  每天一行:snapshot_date, symbol, expiry, atm_iv, dte
fct_oi_snapshot   (append-only)  每天一行/价位:snapshot_date, symbol, expiry, strike, call_oi, put_oi
```

> 把这一条也补回 v1 文档的里程碑 Phase 4:"定时快照"不是 v2 才做,是 v1 就要默默跑起来的。

## 3. 影响面板(优先级 1)

回答你最初那个问题——"期权怎么影响正股"。数据几乎现成:max pain 在 v1 的 `mart_strike_distribution` 已算;GEX 要的 `gamma × OI`,gamma 在 quotes、OI 在 contracts,v1 的 `int_option_chain` 已经 join 好。所以这个面板主要是**展示 + 翻译**,基本不新建管道。

### 3.1 三个子模块

**A. 磁吸位(max pain / 大 OI 价位)**
- 已有数据。翻译:`7/18 那天,$120 堆了最多筹码,临近到期股价容易被吸过去。`

**B. 事件预期(近月 IV 异常抬升)**
- 逻辑:近月 ATM IV 显著高于基准(下一个月到期的 ATM IV,或该票历史常态 IV)→ 市场在为某个事件定价。
- 翻译:`财报前期权定价了 ±12% 的波动,远超平时 → 市场把这次当大事。`

**C. 波动状态(GEX,净 gamma 暴露)**
```
GEX_per_strike = gamma × OI × 100 × S² × 0.01
Net GEX = Σ_calls(GEX) − Σ_puts(GEX)
```
- Net GEX 为正 → 结构倾向压制波动(股价可能黏着小震);为负 → 倾向放大波动。
- 翻译:`当前结构倾向于压制波动,股价可能小幅黏着。`

### 3.2 可信度标签(这个面板的护城河)

每条影响都贴一个标签,**把"定价事实"和"估算猜测"明明白白分开**。市面工具把最不可靠的 GEX 当真理卖,你反过来标清楚——这正是你那个反玄学调性。

| 影响 | 标签 |
|---|---|
| 事件预期(财报前 IV)| `【定价事实 · 最可信】` |
| 磁吸位(max pain)| `【倾向 · 只在临近到期明显】` |
| 情绪倾斜(skew)| `【参考 · 很多是对冲、不是看空】` |
| 波动状态(GEX)| `【估算 · 基于"做市商空 gamma"的假设,谨慎看】` |

> GEX 的正负号取决于"做市商在 call 上空 gamma、在 put 上多 gamma"这个**假设**,不是观测到的事实。务必在标签和 tooltip 里说清楚。

### 3.3 dbt
```
marts/mart_impact.sql
  per (symbol, expiry):
    max_pain_strike, magnet_strikes,
    front_iv, baseline_iv, event_flag,
    net_gex, gex_regime,
    每项附 credibility_tier
```

## 4. 时序特征(优先级 2 · 依赖第 2 节的快照)

### 4.1 内容
- **IV Rank / IV Percentile**:现在期权比过去一年贵还是便宜。
- **墙的演变**:大 OI 价位随时间怎么移动。
- **财报 IV 塌缩(IV crush)**:财报前 IV 抬升、财报后塌缩的经典形态。
- **skew 随时间**:下跌保护需求的冷热变化。

### 4.2 计算(从 `fct_iv_snapshot` 取过去 252 个交易日)
```
IV_Rank       = (IV_now − IV_min_1y) / (IV_max_1y − IV_min_1y) × 100
IV_Percentile = (过去一年 IV 低于今天的天数 / 总天数) × 100
```

### 4.3 翻译(注意:用户不做期权,别说"适合卖权")
- IV Rank 高 → `现在期权很贵,说明市场押注未来波动比平时大得多 → 留意是不是有事要发生。`
- IV crush → `财报一过,期权价格通常会快速回落 — 这是正常现象,不代表风险解除。`

### 4.4 dbt
```
snapshots/fct_iv_snapshot.sql    (append-only, 见第 2 节)
snapshots/fct_oi_snapshot.sql    (append-only)
marts/mart_iv_rank.sql           per (symbol, expiry): iv_now, iv_rank, iv_percentile
marts/mart_wall_evolution.sql    墙的位置随 snapshot_date 的变化
```

## 5. 期限结构面板(优先级 3)

### 5.1 内容
近月 vs 远月的 IV,回答"市场觉得近期有事、还是远期"。近月 IV 明显高于远月 = 有事要发生(财报 / 事件);平滑上行 = 远期不确定性更高。

### 5.2 计算
- 把 v1 的"单一目标到期日"扩展为**拉多个到期日**,各取 ATM IV,以"距到期天数(DTE)"为横轴。
- Backwardation(近月 > 远月)= 近期有事;Contango(远月 > 近月)= 常态。

### 5.3 翻译
`市场觉得未来 3 周内的波动,会比 3 个月后明显更大 — 通常意味着近期有大事(比如财报)。`

### 5.4 dbt
```
marts/mart_term_structure.sql   per (symbol, expiry): dte, atm_iv, shape_flag
```
> 管道改动:抽取层把到期日过滤从"单个"放宽到"最近 N 个"。

## 6. Narrative 层(优先级 4 · 最贵,放最后)

回答"市场情绪上怎么看"。**好消息**:能直接复用你 MacroPulse 那套 LLM schema 抽取(hawk-dove)的思路,把"鹰/鸽"换成"看多/看空 + 关键变化"。**坏消息**:免费新闻情绪质量一般,靠谱的电话会纪要要花钱。所以放最后,别让它拖前三块。

### 6.1 两个子模块
- **新闻情绪**:InsightSentry news 端点 / Finnhub / Alpha Vantage 的 sentiment,聚合成方向 + 强度。
- **电话会语气**:对纪要做 schema 抽取(管理层语气、指引变化),复用 MacroPulse 模式。

### 6.2 翻译
`最近一周关于 MU 的报道整体偏正面,主要围绕 HBM 需求 — 但这是舆论,不是基本面。`

### 6.3 dbt
```
staging/stg_news.sql
marts/mart_sentiment.sql   per (symbol, window): sentiment_score, top_themes, source_quality
```
> 必带 `source_quality` 字段,诚实标注情绪数据本身的可靠度。

## 7. 加分功能

### 7.1 市场预测准不准(回看记分卡)
攒了快照后,对过去的财报 / 到期日,把"市场当时押的预期波动"和"实际走出来的波动"对比:
```
mart_prediction_scorecard
  per past_event: predicted_move_pct, realized_move_pct, hit_or_miss
```
- 翻译:`上次财报,市场押 ±12%,实际走了 ±15% — 市场低估了这次。`
- 价值:把历史变成一个**信任工具**,最贴你诚实的调性。realized move 用标的历史价(yfinance / InsightSentry series)算。

### 7.2 多标的 / 自选股
- 把整条管道在一个 symbol 列表上参数化(dbt var 或 seed `dim_watchlist`)。
- 前端:watchlist 列表 + 单票下钻。

## 10. 风险与注意

- **GEX 的符号是假设、不是观测**:整个 GEX 模块的可信度标签是产品诚信的底线,别为了好看去掉。
- **快照断档**:定时任务挂了而没人发现,会在时序特征里留下永久空洞。freshness 测试 + 告警必须有。
- **财报日历来源**:事件预期、IV crush、预测回看都需要准确的财报日期。确认 InsightSentry 有 events / earnings 端点,否则要接第三方日历。
- **Narrative 数据成本**:免费情绪源质量一般、靠谱纪要要钱;先小范围验证再投入,`source_quality` 字段全程标注。
- **翻译别越界**:用户不做期权,所有时序 / 波动率指标都要翻成"对持有正股的你意味着什么",不要出现"适合卖跨式"这类期权操作建议。
