# 期权透镜 (OptionLens) — 落地文档

> 给只做正股、不懂期权的交易者,把期权市场的信号翻译成人话。
> v1 目标:回答三个问题——这只票大概会落在什么范围、我的目标价市场觉得多大可能、大家把钱押在哪些价位。

---

## 1. 定位与北极星

普通股票交易者看不懂 IV / Greeks / 行权价这套黑话,但期权市场里藏着两样他们真正想要的东西:**一个预估的价格范围**,和**市场对各个价位的概率定价**。这个产品只做一件事——把这些翻译成不带术语的人话。

**北极星测试**:一个不懂期权的人(比如做这个产品之前的我自己)打开任意一屏,不用去 google,就能看懂。每做一个界面都拿这句话量一遍。

**贯穿全程的铁律**:每个期权数字,最后都要落回一句「这对持有正股的你意味着什么」。

## 2. v1 范围(MVP)

三块面板,堆在一个移动端页面里:

| 面板 | 回答的问题 | 一句话产出 |
|---|---|---|
| 预期范围 | 这票大概会到哪? | "到 7/18,大概率落在 $112–$134(±9%)" |
| 问问市场 | 我的目标价有多大可能? | "市场认为收在 $130 上方约 28%" |
| 押注分布 | 大家把钱押在哪个价位? | "$130 挤满赌涨,$120/$110 堆着保护盘" |

**明确砍掉、放进 v2 的**(避免 MVP 失焦):

- 影响面板(磁吸位 / max pain 详解 / GEX)
- 期限结构、skew 详解
- IV Rank / IV Percentile(需要每天攒快照,几周后才有数据)
- Narrative 层(新闻情绪、电话会语气)

> 说明:押注分布面板去掉了「这些条怎么读」和「看专业数字」两个展开层,只保留分布图本身 + 点击查看明细 + 图内一行说明。诚实提醒(OI 是昨收数据、概率是市场定价不是预言)缩成 tooltip 或脚注保留,不另开解释区。

## 3. 技术架构

整套和你现有的 FXLab 栈对齐,基本不引入新东西。

```
InsightSentry REST
   │  (每日定时拉一张当前快照)
   ▼
Python 抽取层  ──►  Parquet 落地 (本地 / S3)
   │
   ▼
DuckDB  ◄──  dbt (dbt-duckdb)  ──►  staging → intermediate → marts
   │
   ▼
FastAPI(可选,薄服务层)
   │
   ▼
Streamlit 移动端三面板
```

| 层 | 选型 | 理由 |
|---|---|---|
| 抽取 | Python(requests)→ Parquet | 跟 FXLab 一致 |
| 仓库 | DuckDB | 数据体量小,本地飞快,跟 dbt 最顺 |
| 转换 | dbt-duckdb | 这是作为 analytics engineer 作品的核心,要 DAG 干净、有测试、有文档 |
| 服务 | FastAPI(可选) | v1 可省略,Streamlit 直连 DuckDB 也行 |
| 展示 | Streamlit | 最快出移动端面板;想做产品感再上 React |
| 调度 | cron / GitHub Actions / Lambda | 跟现有 AWS 架构一致 |

## 4. 数据层:InsightSentry 端点与字段

> 已核对官方文档(insightsentry.com/docs/options)。三个面板要的字段全覆盖,且 Greeks 服务端算好,不用自己反解。

### 4.1 三个端点

| 端点 | 拿什么 | 关键字段 |
|---|---|---|
| `GET /v3/options/quotes` | 报价 + IV + Greeks | `bid_price`、`ask_price`、`implied_volatility`、`bid_iv`、`ask_iv`、`delta`、`gamma`、`theta`、`vega`、`strike_price`、`expiration`、`type` |
| `GET /v3/options/contracts` | 未平仓量 | `open_interest`、`open_interest_date`、`strike_price`、`expiration`、`type`、`code` |
| `GET /v3/symbols/quotes` | 标的现价 | `last_price`(传 `NASDAQ:MU`)|

### 4.2 拉取策略

```bash
# 1. 现价
GET /v3/symbols/quotes?codes=NASDAQ:MU

# 2. 报价链(IV/Greeks/bid-ask)—— 选最近的目标到期日,虚值也带上
GET /v3/options/quotes?code=NASDAQ:MU&from=2026-07-01&to=2026-07-31&range=20

# 3. 合约(OI)—— 同样的过滤条件
GET /v3/options/contracts?code=NASDAQ:MU&from=2026-07-01&to=2026-07-31&range=20
```

- `range=20`:取现价 ±20% 的行权价,保证分布面板两边的虚值墙都拿得到。
- 到期日选择:v1 先锁**最近一个能覆盖下次财报或月度到期**的日期。
- 分页:响应里有 `next_token` 就带着同样的过滤条件翻页。

### 4.3 两个必须设计进架构的坑

1. **OI 和 IV/Greeks 不在同一个端点。** `contracts` 有 OI 没 Greeks;`quotes` 有 Greeks 没 OI。两边都返回期权 code(如 `OPRA:MU260718C130.0`),**按 code join**。这正是 dbt 里"两个 staging 源 → 一张 mart"的活。
2. **OI 是 T+1,历史链不含过期合约。** `open_interest_date` 是前一日收盘;`/series` 历史端点明确不支持已过期期权。含义:OI 面板永远标注"截至昨收",别和当天实时成交量混在一个时间戳;IV Rank 这类时序指标只能靠自己每天落快照往后攒(放 v2)。

## 5. 计算逻辑(翻译核心)

### 5.1 预期范围

```
T  = (到期日 - 今天) / 365
S  = 标的现价
IV = ATM 合约的 implied_volatility   # 取行权价最接近 S 的那个

# 1 个标准差的美元波动(≈ 68% 区间)
expected_move_$ = S * IV * sqrt(T)

band_low  = S - expected_move_$
band_high = S + expected_move_$
pct       = expected_move_$ / S
```

- 校验用:也可以拿 ATM 跨式价 ×0.85 近似 expected move,两者应接近。
- 翻译:`到 {到期日},MU 大概率落在 ${low}–${high} 之间(±{pct}%)`。
- 默认页**不出现**"标准差""68%""IV"任何一个词。

### 5.2 问问市场(目标价概率)

**首选:直接用 delta。** 期权 delta ≈ 风险中性下"到期收在该行权价之上"的概率,InsightSentry 直接给,免计算。

```
输入目标价 X:
  在链上找相邻的两个 call 行权价 K_lo < X < K_hi
  对它们的 delta 做线性插值 → delta_X
  P(收在 X 上方) ≈ delta_X
  P(收在 X 下方) = 1 - delta_X
```

- 翻译:`市场认为 {到期日} 收在 ${X} {上方/下方} 的概率约 {NN}%`,再缀一句人话氛围("机会一般 / 挺有可能 / 有点难")。
- 备选(没 delta 时)对数正态近似:`P(>X) = 1 - Φ( ln(X/S) / (IV·√T) )`。

> 诚实点:这是市场**定价**的概率,不是预言,也不是真实世界概率(delta 是风险中性的)。tooltip 里说清楚。

### 5.3 押注分布

```
# 按行权价聚合(来自 contracts 端点)
per_strike = group_by(strike) -> { call_oi = sum(OI where type=CALL),
                                    put_oi  = sum(OI where type=PUT) }

walls      = top_N(strike by (call_oi + put_oi))
max_pain   = argmin_K( Σ 所有合约在标的=K 到期时的内在价值 )   # 遍历每个候选行权价
pc_ratio   = Σ put_oi / Σ call_oi
```

- 翻译:`押得最多的:${A} 挤满赌涨,${B}/{C} 堆着买保护`。
- 颜色不背锅:**不用红绿涨跌**(你做美股、用户是中国人,A股红涨绿跌、美股反过来,容易搞混)。用青色=赌涨、橙色=买保护,意思全靠文字标签。

## 6. dbt 模型结构

```
models/
├── staging/
│   ├── stg_options_quotes.sql      # quotes 端点:bid/ask, iv, greeks, code, strike, expiry, type
│   ├── stg_options_contracts.sql   # contracts 端点:open_interest, oi_date, code, strike, expiry, type
│   └── stg_underlying.sql          # 标的现价 + 时间戳
├── intermediate/
│   └── int_option_chain.sql        # join quotes ⋈ contracts ON code;解析 code → strike/expiry/type;附 S, T
└── marts/
    ├── mart_expected_move.sql      # 每 (标的, 到期日):ATM IV, expected_move_$, band_low/high, pct
    ├── mart_probability_curve.sql  # 每 (标的, 到期日, 行权价):delta → P(above);供概率计算器插值
    └── mart_strike_distribution.sql# 每 (标的, 到期日, 行权价):call_oi, put_oi, is_wall;加 max_pain, pc_ratio
```

**测试(放 `schema.yml`)**:

- 通用:`not_null` / `unique`(grain 上)/ `relationships`(chain 的 code 必须能对回 staging)。
- freshness:快照时间戳不超过 N 小时。
- 自定义:`open_interest >= 0`;`implied_volatility` 在 (0, 5) 区间;`band_low < band_high`;delta 在 [-1, 1]。

**v2 时序扩展**:加 append-only 的 `fct_iv_snapshot`、`fct_oi_snapshot`,每天追加一行,喂 IV Rank / 墙的演变。

## 7. 翻译层规范

- 维护一张 `dim_glossary`(可做成 dbt **seed**),三列:`术语 → 一句人话 → 对正股交易者意味着什么`。所有面板共用,文案一致。

| 术语 | 一句人话 | 对你意味着 |
|---|---|---|
| IV(隐含波动率)| 市场预期未来波动多大 | IV 高,你的止损要放更远 |
| Expected move | 市场押注的波动范围 | 你的目标位现不现实的尺子 |
| Delta | 到期收在某价之上的概率 | 直接当"市场觉得多大可能" |
| OI(未平仓)| 还没平掉的押注合约数 | 钱压得多的价位值得留意 |
| Max pain | 期权买方整体最受伤的价位 | 临近到期股价的"磁吸位" |

- 三条规则:① 默认页零术语;② 颜色不承担涨跌含义;③ 每个数字落回"对持有正股的你意味着什么"。

## 10. 风险与注意

- **ATM 跨式近似** vs 真实 expected move 有误差,用 IV·√T 为主、跨式价为校验。
- **流动性差的票** bid/ask 很宽,用 mid 估值时要标注"不确定性高";v1 先锁 MU/SPY 这类流动性好的。
- **概率用 delta 是风险中性概率**,不是真实世界概率——产品里诚实说明,别暗示是预言。
- **InsightSentry 套餐与限频**:确认你的 key 覆盖 options 端点,以及每分钟/每日调用上限,定时任务别打爆。
- **OI 滞后**:所有押注分布的数字标注"截至昨收"。
